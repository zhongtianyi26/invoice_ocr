# OCR 服务 API 接口文档

## 接口汇总

| 方法   | 路径              | 说明                           |
| ------ | ----------------- | ------------------------------ |
| `GET`  | `/`               | 根路径，服务状态检查           |
| `POST` | `/ocr/v1/base64`     | 通过 Base64 字符串进行 OCR 识别 |
| `POST` | `/ocr/v1/upload`     | 通过文件上传（multipart/form-data）进行 OCR 识别 |

---

## 1. 根路径

**GET** `/`

成功响应 (200)：

    {
      "message": "PP-OCRv6 OCR Service is running. Use /ocr/base64 (POST form-data with base64_string) or /ocr/upload (POST file) to perform OCR."
    }

---

## 2. Base64 字符串识别

**POST** `/ocr/base64`

### 请求体 (multipart/form-data)

| 字段            | 类型   | 必填 | 说明                                                         |
| --------------- | ------ | ---- | ------------------------------------------------------------ |
| `base64_string` | string | 是   | Base64 编码的图片字符串。可包含 `data:image/...;base64,` 前缀，服务会自动去除。 |

### 成功响应 (200)

    {
      "status": "success",
      "results": ["第一行文字", "第二行文字", ...]
    }

> `results` 为识别出的文字列表，按行顺序排列。

### 失败响应 (400) — 图片解码失败或处理异常

    {
      "detail": "图片处理失败: [具体错误信息]"
    }

### 失败响应 (400) — Base64 解码失败

    {
      "detail": "Invalid base64 string"
    }

---

## 3. 文件上传识别

**POST** `/ocr/upload`

### 请求体 (multipart/form-data)

| 字段   | 类型         | 必填 | 说明                         |
| ------ | ------------ | ---- | ---------------------------- |
| `file` | `UploadFile` | 是   | 图片文件（支持 jpg、png 格式） |

### 成功响应 (200)

    {
      "status": "success",
      "results": ["第一行文字", "第二行文字", ...]
    }

### 失败响应 (400) — 文件类型不支持

    {
      "detail": "仅支持 jpg 或 png 图片"
    }

### 失败响应 (400) — 图片处理异常

    {
      "detail": "图片处理失败: [具体错误信息]"
    }

---

## 错误码汇总

| HTTP 状态码 | 说明           | 常见场景                                     |
| ----------- | -------------- | -------------------------------------------- |
| 200         | 成功           | 正常返回识别结果                             |
| 400         | 请求错误       | 文件类型不支持、Base64 解码失败、图片损坏等  |
| 405         | 方法不允许     | 使用了非 POST 方法访问 `/ocr/base64` 或 `/ocr/upload` |
| 500         | 服务器内部错误 | 模型加载失败、未知异常等（代码中未捕获）     |

> 注意：当前服务未实现 500 错误的自定义响应，若发生内部错误 FastAPI 会返回默认的 500 错误。

---

## 扩展说明

- **模型版本**：服务使用 PP-OCRv6（PaddleOCR 3.x），默认禁用文档方向分类、文本图像矫正、文本行方向分类。
- **性能**：OCR 实例在服务启动时全局初始化，所有请求复用同一实例，避免重复加载模型。
- **返回格式**：目前只返回识别出的纯文本列表，如需置信度和坐标信息，可扩展 `process_image` 函数，解析更完整的 `result` 结构。
